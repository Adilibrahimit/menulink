# 🚀 KO-KO PWA · دليل النشر

## ما في هذا المجلد

```
pwa-starter/
├── koko-menu-v6.html        ← الملف الرئيسي (افتحه في المتصفح)
├── manifest.json            ← يخبر المتصفح أنه تطبيق قابل للتثبيت
├── service-worker.js        ← يجعله يعمل بدون إنترنت
├── icon-72.png              ← أيقونات بأحجام مختلفة
├── icon-96.png
├── icon-144.png
├── icon-152.png
├── icon-180.png             ← لأجهزة Apple
├── icon-192.png             ← Android
├── icon-512.png             ← Android (كبير)
├── icon-maskable-512.png    ← يدعم الأشكال المختلفة لأيقونات أندرويد
├── favicon-32.png
└── favicon-16.png
```

## ⚠️ مهم: PWA يحتاج HTTPS

الـ Service Worker ما يشتغل من ملف محلي. لازم تنشر الملفات على خادم آمن (HTTPS).

## خيارات النشر المجانية

### الأسرع: Manus.space (اللي تستخدمه حالياً)
1. ارفع كل الملفات في `pwa-starter/` كمجلد واحد
2. تأكد إنه `koko-menu-v6.html` هو ملف الـ index
3. الرابط الناتج هو الـ PWA

### بديل قوي: Netlify
1. أنشئ حساب على [netlify.com](https://www.netlify.com) (مجاني)
2. سحب وإفلات (drag & drop) للمجلد كاملاً
3. تحصل على رابط `xxxxx.netlify.app` فوراً
4. اربط نطاق خاص لاحقاً (`menu.koko.sa`) مجاناً

### بديل آخر: Vercel
1. أنشئ حساب على [vercel.com](https://vercel.com)
2. ارفع المجلد عبر CLI أو واجهة الويب
3. نفس الفكرة

## كيف يثبّت الزبون التطبيق؟

### Android (Chrome):
1. يفتح الرابط
2. يظهر له **شريط KO-KO الأحمر** بعد ٥ ثواني → يضغط "تثبيت"
3. أو من قائمة Chrome → "تثبيت التطبيق"
4. الأيقونة تطلع على الشاشة الرئيسية مثل أي تطبيق

### iPhone (Safari):
1. يفتح الرابط
2. يضغط زر المشاركة (□↑)
3. "إضافة إلى الشاشة الرئيسية"
4. الأيقونة تطلع على الشاشة

## ✏️ كيف تخصّصه لعميل آخر

في الوقت الحالي (قبل بناء النظام المتقدم)، لإنشاء PWA لعميل جديد:

1. **استبدل الأيقونات:** ولّد أيقونات بشعار العميل بنفس الأحجام
2. **عدّل `manifest.json`:**
   - `name`: اسم المطعم
   - `short_name`: اسم مختصر
   - `theme_color`: لون العميل
   - `background_color`: لون الخلفية
3. **عدّل `koko-menu-v6.html`:**
   - بيانات `MENU` (الأصناف والأسعار)
   - `CONFIG.restaurantName` و `whatsappPhone`
   - الألوان في الـ CSS variables أعلى الملف
   - اللوجو (SVG داخل الـ HTML)
4. **عدّل `service-worker.js`:**
   - غيّر `VERSION` (مهم! بدونه المتصفح يستخدم النسخة القديمة من الكاش)

> هذه العملية تأخذ ساعتين تقريباً لكل عميل. الحل الدائم (المرحلة ٢ في ROADMAP) هو لوحة تحكم تسوي كل ذلك في ٣٠ دقيقة.

## 🧪 اختبار محلي قبل النشر

PWA ما يشتغل من `file://` لكن تقدر تختبره محلياً بخادم بسيط:

```bash
cd pwa-starter
python3 -m http.server 8080
# افتح http://localhost:8080/koko-menu-v6.html
```

في كروم: F12 → Application → Service Workers → تأكد إنه نُسجِّل.

## 📊 ما اللي يصير بعد ما تتثبت؟

| السلوك | قبل PWA (HTML عادي) | بعد PWA |
|------|----------------|---------|
| فتح الموقع بدون إنترنت | ❌ لا يعمل | ✅ يفتح من الكاش |
| أيقونة على الشاشة | ❌ يحتاج bookmark | ✅ كأي تطبيق |
| سرعة التحميل (مرة ثانية) | ~٢-٤ ثواني | ~٠.٥ ثانية |
| واجهة بدون شريط المتصفح | ❌ | ✅ |
| إشعارات Push | ❌ | ✅ (يحتاج OneSignal) |

## ❓ مشاكل شائعة

**"الـ install prompt ما يظهر"**
- لازم HTTPS
- لازم تفتح الموقع مرتين على الأقل بفارق وقت
- لازم تكون عبر شبكة موثوقة (مش proxy)
- iPhone ما يدعم prompt تلقائي - الزبون يضيفه يدوياً

**"الخريطة ما تطلع"**
- الخريطة تحتاج إنترنت (تيلز OpenStreetMap)
- المنيو وحدها تشتغل بدون إنترنت

**"الواتساب ما يفتح من التطبيق المثبّت"**
- يجب أن يكون واتساب مثبّت على الجهاز
- نظام التشغيل يفتحه تلقائياً عند الضغط على رابط wa.me
